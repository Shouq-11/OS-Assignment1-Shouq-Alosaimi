## Question 1: What did you learn about multithreading?

**Your Answer:**

[Write your answer here. Discuss specific concepts like thread creation, thread states, how threads execute concurrently, what surprised you, etc.]

I discovered that multithreading enables a program to use threads inside the same process to execute several tasks concurrently. Because threads share memory, communication between them is quicker and more effective than when separate processes are used. Additionally, I learnt about the many thread states—New, Runnable, Running, Waiting, and Terminated—and how they change when a thread is being executed. Through this project, I gained an understanding of how Round-Robin scheduling assigns a fixed time quantum to each thread. Additionally, I discovered how context flipping occurs when a thread exits the CPU and a new one begins. In general, multithreading enhances software responsiveness and performance.

## Question 4: How can you apply multithreading concepts in real-world applications?

**Your Answer:**

[Give specific examples from real applications you use (web browsers, games, mobile apps, etc.). Explain why threads are useful in those scenarios. Connect to what you learned in this assignment.]

Numerous real-world applications, including web servers, mobile apps, and games, can use multithreading. A web server, for instance, uses several threads to quickly process numerous user requests at once. Threads are used in mobile applications to carry out background operations, such as data downloads, while the user is continuously interacting with the program. Threads in games can manage multiple activities at once, including physics, graphics, and human input. The application becomes more responsive and performs better as a result. In general, multithreading makes programs operate more quickly and effectively manage several tasks.
